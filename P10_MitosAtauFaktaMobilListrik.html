Project Mobile Game Mitos Atau Fakta Mobil Listrik

An educational mobile game that explores the facts and myths about electric cars with an interactive dialogue style.

Status : Finished

Platform : Android

Technology : Unity Game Engine, Blender 3D

In this work, I have a client who requested an educational game about electric cars for Android. During this project, I extensively researched Unity optimization techniques and implemented effective teamwork workflows. From this work i can understand workflow using github more.

Core Mechanics

The gameplay of Mitos atau Fakta Mobil Listrik

This game is about reading stories and deciding if they are a myth or fact. There are no penalties for the player's answers, and they can continue playing until they complete the game. It's an educational game with less focus on entertainment.

What have I worked on?

Please keep in mind that these explanations is simply me sharing what I did to get the result. I believe there's room for improvement in the code and system, but currently, I don't have the skills and time to make those optimizations.

  1. Created a tool for Unity Timeline to create custom dialogue clip

    2. The Unity Timeline of the game

    The Unity Timeline acts as a place that can contain different tracks, where each track can contain different clips (Just like Premiere Pro or any other timelines). As a developer, I wanted to optimize the game and make it more efficient, so I developed a tool for the dialogue system within the Unity Timeline.

    The tool makes it a lot easier to manage, as it can provide more visibility and controls of the dialogue in the Timeline instead of dealing with a bunch of buttons and manually set the duration for each dialogue panel in the Inspector tab. I also considered our 3D artist on the team and made sure the tool is user-friendly for them.

    Demo of the dialogue system tool

    To create the dialogue system tool within Unity Timeline, I found a solution by editing the Timeline script. I began by researching how to achieve this functionality and came across a similar template in the Unity Timeline demo project.

    After downloading the project to study it, I imported the script into my project and proceeded to make modifications by adding "Fakta" and "Mitos" buttons, as well as a finished button to achieve the result I want for the dialogue clip.

    You can find the Unity Timeline project demo in the Unity's blog by visiting this link https://blog.unity.com/technology/creative-scripting-for-timeline.

    Now, let's dive deeper into the technical side. In total, there are 7 scripts, including the Singleton script, that contribute to achieving this result. You can see how they depend on each other in the image below.

    Diagram of the dialogue system tool

    As you can see the GameManager, UIManager, and InputManager use the Singleton class as an implementation of the Singleton pattern to create their instances. So, whenever other classes need to communicate with them, they just need to use the Singleton instance. This makes things super easy because you can access them with just one line of code, without having to pass their references. For example this is how to access the GameManager instance using singleton class:

    GameManager.Instance.ResumeTimeline();

    example code to access GameManager Instance

    Let's see the breakdown of each step of the pause process:

    1. The DialogueBehaviour asks the DialogueClip if there are any changes on the clip.

      2. First the DialogueBehaviour detects if there are any changes on the played DialogueClip by assigning DialogueBehaviour as the behaviour of the DialogueClip, you can see it in this code

      public DialogueBehaviour template = new DialogueBehaviour ();

      The DialogueClip creates an instance of the DialogueBehaviour class and assigns it to the 'template' variable.

      public override Playable CreatePlayable (PlayableGraph graph, GameObject owner)
    {
        var playable = ScriptPlayable<DialogueBehaviour>.Create (graph, template);
        return playable;
    }

      example code to access GameManager Instance

    2. The DialogueClip asks the DialogueTrack if there are any updates.
    3. The DialogueTrack gives an update to the DialogueClip that there haven't been any changes made on this dialogue track.
    4. The DialogueClip gives an update to the DialogueBehaviour that there is a change for this clip. The characterName is set to "Tosa," hasPause boolean is enabled, and the dialogueLine has the text "Test" and a dialogueFontSize of 30.
    5. The DialogueBehaviour gives an update to the GameManager to execute PauseTimeline(), and also gives an update to the UIManager to change the characterName, dialogueLine, and dialogueFontSize through the SetDialogue() method.
    6. The UIManager reads the update from the DialogueBehaviour and executes the SetDialogue() method.
    7. The GameManager reads the update from the DialogueBehaviour and executes the PauseTimeline() method.
    8. The GameManager gives an update to the InputManager that the game is now in dialogue mode.
    9. The InputManager reads the update from the GameManager.
    10. The GameManager gives an update to the UIManager to add an onClick event for the nextSentenceButton and the GameManager's ResumeTimeline() method.
    11. The UIManager reads the update from the InputManager and proceeds to add onClick.AddListener on the buttons.
    12. The UIManager gives an update to the GameManager to add the ResumeTimeline() method to the onClick event.
    13. The GameManager reads the update from the UIManager and then adds ResumeTimeline() to the onClick event.

  2. Optimized the game using various Unity optimization techniques

    3. When it comes to optimizing the game, it can be a bit tricky. In this project, I implemented some basic optimization techniques, such as providing quality options, using multiple canvases, reducing draw calls, disabling unused event listeners, using GPU instancing, and using light probes with baked lighting.
